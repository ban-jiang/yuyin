# 余音项目交接摘要

## 产品定位

"余音"是一款诗词视觉卡创作工具。用户用自然语言搜索喜欢的诗人、词人、作品、名句或情绪，从 AI 候选中确认作品与核心句，再选择艺术模板并导出卡片。

核心原则：AI 负责寻找和整理内容，用户负责选择，前端模板负责稳定排版。AI 不直接生成带中文文字的图片，也不自由控制文字坐标。

## 本轮关键决策

- 产品先聚焦古诗词和古文，不做通用长文章工具。
- 不预先建设庞大诗词库，由 DeepSeek 理解用户意图并返回候选。
- 模型原文存在幻觉风险，因此候选标记为"待核验"，用户可以编辑原文。
- 用户直接浏览全部模板，不由 AI 推荐模板。
- 句子数量不向用户暴露，由 AI 与前端根据空间自动决定。
- 所有文字使用固定分区网格和真实尺寸测量，禁止重叠。

## 已实现功能

### 内容流程（v0.1 → v0.2 重大变更）

**旧流程（v0.1）：** 搜索 → 单选一首 → 逐句勾选 → 生成卡片

**新流程（v0.2）：** 搜索 → 勾选多首诗词 → 点击「AI 策展选句」→ `/api/curate` 从各首均衡选句 → 编辑确认 → 生成卡片

1. 自然语言输入：支持诗人、作品、名句和情绪描述。
2. `POST /api/search` 调用 DeepSeek，返回结构化候选作品。
3. 候选卡片显示**红色徽标**（⚠ AI 候选 · 原文待核验），提醒用户核实。
4. 未配置 `DEEPSEEK_API_KEY` 时自动使用本地演示数据。
5. 用户勾选多首诗词后点「AI 策展选句」，策展结果标注每句来源。
6. 策展结果默认只读，点击「编辑原文」解锁后可修改。
7. 句子数量由 AI 策展决定（5-9 句），用户可编辑后确认。
8. 生成自定义卡片后，右侧隐藏演示切换区，显示当前作品和"重新选择诗词"。

### 卡片编辑

- 演示内容集：毛泽东、苏轼、辛弃疾。
- 简体/繁体切换，覆盖正文、来源、主题字和印章。
- 四套背景：纸白、煤墨、铜绿、靛夜。
- 署名统一位于左下角，印章统一位于右下角。
- 大主题字完整居中、极低透明度显示，不被色块遮住或画布裁切。
- 文字块在模板/背景/简繁切换后自动重新测量；仅缩小发生溢出的句子。

### 当前模板（7 套）

1. 编辑留白 — 模块网格 · 单一视觉焦点
2. 宋刻手卷 — 版心界栏 · 纵向行气
3. 现代构成 — 几何分区 · 黑红对照
4. 活字套印 — 双色油墨 · 压印边缘
5. 夜读笺 — 深纸 · 金墨 · 静谧低对比
6. 印象拼贴 — 后现代色块 · 半透明叠印
7. 字阵残章 — 横竖混排 · 多色层级

> 已删除：碑拓余痕（rubbing）、题跋册页（album）—— 视觉不理想。

## 技术结构

```
寻找项目/
├── index.html          # 全部前端（模板 CSS + 渲染 + 选择流程）
├── server.js           # 本地开发 HTTP 服务
├── api/
│   ├── search.js       # Vercel serverless: /api/search
│   └── curate.js       # Vercel serverless: /api/curate
├── src/
│   └── api.js          # 前端 API 封装（searchPoetry + curatePoetry）
├── vercel.json         # Vercel 部署配置
├── package.json        # Node 24.x 引擎
├── .env.example        # 环境变量模板
├── DEEPSEEK_CARD_SPEC.md
└── README.md
```

- 零第三方依赖，纯 Node.js + 原生前端。
- 本地开发用 `node server.js`，线上用 `api/` 目录下的 serverless 函数。
- 同一套代码，本地和线上均通。

- 已部署至 Vercel：**https://yuyin-gray.vercel.app/**

## 已知问题与规避

| 问题 | 表现 | 已采取措施 |
|------|------|-----------|
| **AI 作者混淆** | 搜"王安石"出现毛泽东诗词 | 搜索 prompt 强约束 + 指定作者服务端精确过滤 + 不再用无关演示数据兜底 |
| **AI 策展返回非原文句子** | 策展结果包含原文之外的句子 | 服务端规范化比对 + 宽松 fallback |
| **活字套印底部框重叠** | letterpress q6/q7 视觉重叠 | 对齐列边界为 4 列分割 + 增大 gap |
| **字阵残章 13 句版** | 过于拥挤凌乱 | **已回滚**，保持原始 9 句设计 |

- 按钮现在在 `data-composition="a"` 与 `data-composition="b"` 间切换，并显示当前 A/B 状态；切换模板时当前构图状态继续生效。

- 根据用户提供的夜间诗词灯光装置参考，首页背景进一步调整为“深夜林间诗灯”：保留原定 9 句诗词一字不改，改为全部竖排、高低错落的金色发光文字；使用多层 `text-shadow` 形成暖黄灯光，CSS 渐变生成抽象枝线、垂落灯串和散点光粒。原 SVG 纹理降亮并以 `screen` 混合，作为暗部树皮/空气颗粒质感。桌面和 390px 手机端分别定义九列位置，背景仍不拦截任何交互。
- 首页背景最终收敛为“淡黑月夜”：去除强烈灯串、枝线和霓虹光晕，以淡黑灰径向渐变为底，右上方设置克制的冷灰月光区域，仅保留极弱颗粒。9 句原文继续保留，恢复横竖错落排布，统一使用卡片正文 `var(--serif)`（汇文明朝体/宋体回退），文字为低透明度淡金色。手机端单独降低字号与透明度，并限制长横句宽度。
- 首页背景随后进一步精简：完全删除 9 句装饰诗词的 DOM，仅保留淡黑月夜。在画面底部新增纯 CSS 河流，以低对比度深灰青渐变、横向细波纹和不规则月光反射构成；桌面河面约占底部 31%，手机端约占 24%，不进入搜索框主要操作区域，继续保持 `pointer-events:none`。
- 新增“受约束的句子微调”第一阶段：仅在用户生成自定义卡片后显示。用户点击卡片句子进行选择，可用上下箭头与相邻句子交换固定网格槽位，可选正文/强调/柔和三种模板安全色，以及紧凑/标准/强调三级字号。字号调整仍经过 `fitQuotes()` 溢出检测，必要时自动缩小；不开放自由拖拽，避免重叠。支持一键恢复生成时的默认顺序、颜色和字号，切换到句数较少的模板时会自动清除不可见句子的选中状态。
- 新增编辑历史与本地草稿：句子换位、颜色、字号、模板、纸墨和 A/B 构图变化前都会保存快照，最多支持 20 步“撤销”。自定义作品及其句子样式、模板、背景、构图、简繁状态通过 `localStorage` 保存在当前浏览器，首页出现“恢复上次作品”；草稿损坏时自动清除，不上传用户文本到额外服务。生成新卡片时重置撤销栈并覆盖最近草稿。
- 句子微调色板更新为白色、浅绿色、深绿色、浅灰色、深黑色五种明确颜色；新增“常规/加粗”字重切换。字重与颜色一样会随句子换位、撤销和本地草稿保存。选中句子的编辑轮廓可通过“完成”按钮或点击卡片空白处取消；进入“截图保存”预览时也会自动取消选中，确保轮廓不进入成品。
- 新增“用户粘贴古风歌词”入口。用户填写作品名、词作者/演唱者，按每行一句粘贴自己有权使用的歌词并确认权限；`POST /api/lyrics` 调用 DeepSeek 从用户提供的原文中原样提取 8-15 条候选，服务端用精确匹配过滤改写、拼接或新增内容，模型结果不足时回退到原文行。用户再自行勾选 4-9 句生成卡片，最多选择 9 句。该接口同时提供 Vercel `api/lyrics.js`、Netlify Function 和本地 Node 路由。歌词卡进入现有自定义卡片系统，支持七套模板、A/B 构图、背景、简繁、句子颜色/字号/字重、撤销和本地草稿；生成新歌词卡或诗词卡时会清除上一张作品遗留的单句样式。
- 古代内容检索范围从“诗词”扩展为“中国古代文学”，支持诗、词、曲、赋、散文、序、记等古文。首页改问“你喜欢哪位古代诗人、词人或文学家”，候选新增 `genre` 体裁标签。用户明确同时要求诗词和古文时，DeepSeek prompt 强制两类至少各返回一篇，服务端再次检查候选中同时存在诗/词/曲与古文/赋/序/记，否则拒绝不完整结果并提示重试；例如“苏轼的诗词和古文”应包含诗词及《赤壁赋》等古文。作者硬过滤名单扩展到常见古代文学家。策展 prompt 同步支持诗词与古文混选，并要求两类都有原句入选。
- 首页“粘贴古风歌词”入口升级为更大、更醒目的独立视觉按钮，文案改为“粘贴古风歌词，AI帮你选句”。

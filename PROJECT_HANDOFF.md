# 余音项目交接摘要

## 产品定位

"余音"是一款诗词视觉卡创作工具。用户用自然语言搜索喜欢的诗人、词人、作品、名句或情绪，从 AI 候选中确认作品与核心句，再选择艺术模板并导出卡片。

核心原则：AI 负责寻找和整理内容，用户负责选择，前端模板负责稳定排版。AI 不直接生成带中文文字的图片，也不自由控制文字坐标。

## 本轮关键决策

- 产品先聚焦古诗词和古文，不做通用长文章工具。
- 不预先建设庞大诗词库，由 DeepSeek 理解用户意图并返回候选。
- 模型原文存在幻觉风险，因此候选标记为"待核验"，用户可以编辑原文。
- 用户直接浏览全部模板，不由 AI 推荐模板。
- 句子数量不向用户暴露，由 AI 与前端根据空间自动决定。
- 所有文字使用固定分区网格和真实尺寸测量，禁止重叠。

## 已实现功能

### 内容流程（v0.1 → v0.2 重大变更）

**旧流程（v0.1）：** 搜索 → 单选一首 → 逐句勾选 → 生成卡片

**新流程（v0.2）：** 搜索 → 勾选多首诗词 → 点击「AI 策展选句」→ `/api/curate` 从各首均衡选句 → 编辑确认 → 生成卡片

1. 自然语言输入：支持诗人、作品、名句和情绪描述。
2. `POST /api/search` 调用 DeepSeek，返回结构化候选作品。
3. 候选卡片显示**红色徽标**（⚠ AI 候选 · 原文待核验），提醒用户核实。
4. 未配置 `DEEPSEEK_API_KEY` 时自动使用本地演示数据。
5. 用户勾选多首诗词后点「AI 策展选句」，策展结果标注每句来源。
6. 策展结果默认只读，点击「编辑原文」解锁后可修改。
7. 句子数量由 AI 策展决定（5-9 句），用户可编辑后确认。
8. 生成自定义卡片后，右侧隐藏演示切换区，显示当前作品和"重新选择诗词"。

### 卡片编辑

- 演示内容集：毛泽东、苏轼、辛弃疾。
- 简体/繁体切换，覆盖正文、来源、主题字和印章。
- 四套背景：纸白、煤墨、铜绿、靛夜。
- 署名统一位于左下角，印章统一位于右下角。
- 大主题字完整居中、极低透明度显示，不被色块遮住或画布裁切。
- 文字块在模板/背景/简繁切换后自动重新测量；仅缩小发生溢出的句子。

### 当前模板（7 套）

1. 编辑留白 — 模块网格 · 单一视觉焦点
2. 宋刻手卷 — 版心界栏 · 纵向行气
3. 现代构成 — 几何分区 · 黑红对照
4. 活字套印 — 双色油墨 · 压印边缘
5. 夜读笺 — 深纸 · 金墨 · 静谧低对比
6. 印象拼贴 — 后现代色块 · 半透明叠印
7. 字阵残章 — 横竖混排 · 多色层级

> 已删除：碑拓余痕（rubbing）、题跋册页（album）—— 视觉不理想。

## 技术结构

```
寻找项目/
├── index.html          # 全部前端（模板 CSS + 渲染 + 选择流程）
├── server.js           # 本地开发 HTTP 服务
├── api/
│   ├── search.js       # Vercel serverless: /api/search
│   └── curate.js       # Vercel serverless: /api/curate
├── src/
│   └── api.js          # 前端 API 封装（searchPoetry + curatePoetry）
├── vercel.json         # Vercel 部署配置
├── package.json        # Node 24.x 引擎
├── .env.example        # 环境变量模板
├── DEEPSEEK_CARD_SPEC.md
└── README.md
```

- 零第三方依赖，纯 Node.js + 原生前端。
- 本地开发用 `node server.js`，线上用 `api/` 目录下的 serverless 函数。
- 同一套代码，本地和线上均通。

## 线上部署

- 已部署至 Vercel：**https://yuyin-gray.vercel.app/**
- GitHub 仓库：**https://github.com/ban-jiang/yuyin**
- 推送代码后 Vercel 自动部署。
- 环境变量 `DEEPSEEK_API_KEY` 需在 Vercel Settings 配置。
- 部署关键：Framework Preset 设为 **Other**，Build/Output/Install 全部留空。
- Node.js 引擎版本：`24.x`（Vercel 2026 年已弃用 18.x）。

## 本地运行

```powershell
node server.js
# 打开 http://127.0.0.1:4174/

# 启用 DeepSeek:
$env:DEEPSEEK_API_KEY="你的密钥"

```

## 已知问题与规避

| 问题 | 表现 | 已采取措施 |
|------|------|-----------|
| **AI 作者混淆** | 搜"王安石"出现毛泽东诗词 | 搜索 prompt 强约束 + 指定作者服务端精确过滤 + 不再用无关演示数据兜底 |
| **AI 策展返回非原文句子** | 策展结果包含原文之外的句子 | 服务端规范化比对 + 宽松 fallback |
| **活字套印底部框重叠** | letterpress q6/q7 视觉重叠 | 对齐列边界为 4 列分割 + 增大 gap |
| **字阵残章 13 句版** | 过于拥挤凌乱 | **已回滚**，保持原始 9 句设计 |

- 策展句数统一为 7～9 句：DeepSeek prompt 改为 7～9 句，服务端在模型少给时从用户已选作品原句中去重补足到至少 7 句。

- 搜索结果增加“当前限定作者”提示；无结果时明确说明不会用其他作者作品替代；接口失败时提供“重新尝试”。

- 自定义作品生成后继续隐藏演示诗人选择，只保留“重新选择诗词”；同时修复自定义作品点击“换一个构图”可能失效的问题。
- 新增 Netlify 部署支持：`netlify.toml`、`netlify/functions/search.js`、`netlify/functions/curate.js`。前端继续请求 `/api/search` 与 `/api/curate`，由 Netlify redirects 转发到 Functions。
- Vercel 与 Netlify 可并行部署；两端都需要单独配置环境变量 `DEEPSEEK_API_KEY`，密钥不得提交到 GitHub。
- 入口搜索页增加低透明度诗句背景，选取《诗经》、汉魏、晋、唐、宋、元、清等时期名句，以横排、竖排和轻微角度变化形成文学氛围；背景不响应点击，不影响搜索操作，手机端自动减少诗句数量。
- 删除无实际价值的“印刷偏移”控制项及事件逻辑。
- 修复普通模板仍出现 9 句的问题：增加 CSS 硬约束，除 `glyphfield`（字阵残章）外，所有模板强制隐藏 q8/q9；原有 JavaScript 7/9 句规则继续作为第一层约束。
- 查明卡片底部出现毛泽东诗句的原因：模板初始 DOM 使用毛泽东诗词作为演示占位，旧逻辑只隐藏未用句子而未清空文本。现在自定义作品渲染时会清空所有未使用占位文字，并用 `[hidden]` 强制隐藏，避免旧诗句被模板 CSS 重新显示。
- “换个构图”升级为真实 A/B 双构图，不再只是随机更换主题字和强调色。七种模板均新增独立 B 网格：编辑留白采用瑞士式偏心主块与窄竖边注；宋刻手卷改为三栏纵排重心；现代构成采用包豪斯式尺度碰撞；活字套印采用阶梯错位字块；夜读笺采用偏心留白与纵向侧句；印象拼贴采用不对称色面节奏；字阵残章重新组织横竖字簇。
- 双构图参考公开的 Swiss Style / International Typographic Style 原则：模块网格、非对称布局、左对齐和负空间；同时吸收包豪斯几何尺度对比及日本现代书籍设计的偏心留白和纵横阅读方向切换。实现只使用固定网格单元，不使用随机绝对坐标。
- 按钮现在在 `data-composition="a"` 与 `data-composition="b"` 间切换，并显示当前 A/B 状态；切换模板时当前构图状态继续生效。
- 本地端到端验证：完成搜索、选诗、策展、生成后，逐一检查七种 B 构图。所有普通模板均显示 7 句，七种模板均无文字溢出、无句块几何重叠；390×844 手机视口无横向溢出，编辑器保持可用。

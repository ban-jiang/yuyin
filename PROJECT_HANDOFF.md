# 余音项目交接摘要

## 产品定位

"余音"是一款诗词视觉卡创作工具。用户用自然语言搜索喜欢的诗人、词人、作品、名句或情绪，从 AI 候选中确认作品与核心句，再选择艺术模板并导出卡片。

核心原则：AI 负责寻找和整理内容，用户负责选择，前端模板负责稳定排版。AI 不直接生成带中文文字的图片，也不自由控制文字坐标。

## 本轮关键决策

- 产品先聚焦古诗词和古文，不做通用长文章工具。
- 不预先建设庞大诗词库，由 DeepSeek 理解用户意图并返回候选。
- 模型原文存在幻觉风险，因此候选标记为"待核验"，用户可以编辑原文。
- 用户直接浏览全部模板，不由 AI 推荐模板。
- 句子数量不向用户暴露，由 AI 与前端根据空间自动决定。
- 所有文字使用固定分区网格和真实尺寸测量，禁止重叠。

## 已实现功能

### 内容流程（v0.1 → v0.2 重大变更）

**旧流程（v0.1）：** 搜索 → 单选一首 → 逐句勾选 → 生成卡片

**新流程（v0.2）：** 搜索 → 勾选多首诗词 → 点击「AI 策展选句」→ `/api/curate` 从各首均衡选句 → 编辑确认 → 生成卡片

1. 自然语言输入：支持诗人、作品、名句和情绪描述。
2. `POST /api/search` 调用 DeepSeek，返回结构化候选作品。
3. 候选卡片显示**红色徽标**（⚠ AI 候选 · 原文待核验），提醒用户核实。
4. 未配置 `DEEPSEEK_API_KEY` 时自动使用本地演示数据。
5. 用户勾选多首诗词后点「AI 策展选句」，策展结果标注每句来源。
6. 策展结果默认只读，点击「编辑原文」解锁后可修改。
7. 句子数量由 AI 策展决定（5-9 句），用户可编辑后确认。
8. 生成自定义卡片后，右侧隐藏演示切换区，显示当前作品和"重新选择诗词"。

### 卡片编辑

- 演示内容集：毛泽东、苏轼、辛弃疾。
- 简体/繁体切换，覆盖正文、来源、主题字和印章。
- 四套背景：纸白、煤墨、铜绿、靛夜。
- 署名统一位于左下角，印章统一位于右下角。
- 大主题字完整居中、极低透明度显示，不被色块遮住或画布裁切。
- 文字块在模板/背景/简繁切换后自动重新测量；仅缩小发生溢出的句子。

### 当前模板（7 套）

1. 编辑留白 — 模块网格 · 单一视觉焦点
2. 宋刻手卷 — 版心界栏 · 纵向行气
3. 现代构成 — 几何分区 · 黑红对照
4. 活字套印 — 双色油墨 · 压印边缘
5. 夜读笺 — 深纸 · 金墨 · 静谧低对比
6. 印象拼贴 — 后现代色块 · 半透明叠印
7. 字阵残章 — 横竖混排 · 多色层级

> 已删除：碑拓余痕（rubbing）、题跋册页（album）—— 视觉不理想。

## 技术结构

```
寻找项目/
├── index.html          # 页面语义结构与资源入口
├── styles/
│   └── app.css         # 全局界面、卡片模板、响应式与艺术背景
├── server.js           # 本地开发 HTTP 服务
├── api/
│   ├── _security.js    # 限流、超时与热实例缓存
│   ├── search.js       # Vercel serverless: /api/search
│   ├── curate.js       # Vercel serverless: /api/curate
│   └── lyrics.js       # Vercel serverless: /api/lyrics
├── src/
│   ├── poster.js       # 卡片数据、正文渲染与字号适配
│   ├── storage.js      # 草稿、撤销与最近作品
│   ├── editor.js       # 单句选择、换位、颜色、字号与字重
│   ├── controls.js     # 模板、构图、背景、画幅与移动端标签
│   ├── discovery.js    # 古代文学搜索、策展与歌词入口
│   ├── app.js          # 最小启动入口与版本标记
│   └── api.js          # 前端 API 请求封装
├── vercel.json         # Vercel 部署配置
├── package.json        # Node 24.x 引擎
├── .env.example        # 环境变量模板
├── DEEPSEEK_CARD_SPEC.md
└── README.md
```

- 零第三方依赖，纯 Node.js + 原生前端。
- 本地开发用 `node server.js`，线上用 `api/` 目录下的 serverless 函数。
- 同一套代码，本地和线上均通。

## 已知问题与规避

| 问题 | 表现 | 已采取措施 |
|------|------|-----------|
| **AI 作者混淆** | 搜"王安石"出现毛泽东诗词 | 搜索 prompt 强约束 + 指定作者服务端精确过滤 + 不再用无关演示数据兜底 |
| **AI 策展返回非原文句子** | 策展结果包含原文之外的句子 | 服务端规范化比对 + 宽松 fallback |
| **活字套印底部框重叠** | letterpress q6/q7 视觉重叠 | 对齐列边界为 4 列分割 + 增大 gap |
| **字阵残章 13 句版** | 过于拥挤凌乱 | **已回滚**，保持原始 9 句设计 |

## v0.3 基础设施与移动端（2026-07-13）

- 新增共享接口安全模块 `api/_security.js`：按客户端 IP 每分钟限制 12 次 AI 请求，DeepSeek 请求 25 秒超时，搜索结果在热实例内缓存 10 分钟并限制最多 200 条。Vercel 与 Netlify Functions 共用该模块，Netlify 适配器会传递请求头；本地 Node 服务另设每分钟 30 次的开发限制。
- 当前限流和缓存为进程内实现，只适合作为第一层保护；Serverless 冷启动或多实例之间不共享状态。正式大规模运营时应迁移到 Upstash Redis、Cloudflare KV 或托管数据库实现全局限流、每日额度和持久缓存。

- 截图工作流新增 3:4 社交分享与 A4 竖版两种画幅，切换后调用 `fitQuotes()` 重新检测溢出。编辑器可显示 4% 内缩的安全边距参考线，进入截图预览时自动隐藏。画幅与参考线状态均进入撤销、本地草稿和最近作品历史；手机端归入“版式”标签。

## v0.6 诗泉完整原文与全文自选（2026-07-14）

- 新增共享来源模块 `api/_poetry-source.js`，本地 Node、Vercel 与 Netlify 共用。DeepSeek仍负责理解用户意图和返回候选元数据；对于诗、词、曲，服务端再调用 `https://poetry.palemoky.com/api/search` 获取完整原文。
- 诗泉是全文模糊搜索，不能直接使用第一条结果。解析器优先取 DeepSeek 候选中的代表性分句反查，再按作者完全一致、标题核心一致和正文重合度评分；只有达到阈值才替换原句并标记 `sourceStatus: poetry-api`，否则保留 `model-unverified`，避免同名作品和正文提及造成误匹配。
- 来源查询设置9秒超时和24小时热实例缓存；诗泉不可用、超时或没有可靠匹配时静默回退到AI候选，不阻断搜索。可用 `POETRY_API_BASE_URL` 更换服务地址，默认无需配置API Key。
- 搜索候选新增“诗泉原文 · N句”绿色来源标记；未匹配候选显示“AI候选 · 原文待核验”。AI策展页在全部作品均已匹配时显示“选自诗泉原文”。
- “自己选句”升级为“全文自选”：已匹配作品展示诗泉返回的全部 `content`，作品按折叠区分组并显示来源和总句数；增加全文关键词搜索与“只看已选”。最终选择仍严格限制4-9句，普通模板显示前7句，字阵残章最多9句。
- `/api/curate` 不再把每篇作品截断为9句。最多接收来源层240行；超过120行时从全文均匀取样120行交给模型，避免只看开头，同时限制单行长度。卡片输出仍限制7-9句。Local Node请求体上限由20KB提高到120KB以容纳完整诗词。
- 实测来源匹配：苏轼《定风波·莫听穿林打叶声》匹配诗泉《定风波》11句；辛弃疾《青玉案·元夕》匹配《青玉案》8句；李白《将进酒》匹配同名作品12句。作者与标题均正确。
- 契约测试：模拟DeepSeek候选后，`/api/search` 返回12句诗泉全文；`/api/curate` 收到全部12句并能选中第12句，确认中间链路不再截回9句。全部服务端和浏览器脚本通过语法及共同编译检查。
- 当前推荐 `DEEPSEEK_MODEL=deepseek-chat`，用于意图识别、候选和策展。古文双模型阶段预留 `DEEPSEEK_REVIEW_MODEL=deepseek-reasoner`；同一模型二次自审不能替代程序差异比较和用户确认。

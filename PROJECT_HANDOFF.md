# 余音项目交接摘要

## 产品定位

“余音”是一款诗词视觉卡创作工具。用户用自然语言搜索喜欢的诗人、词人、作品、名句或情绪，从 AI 候选中确认作品与核心句，再选择艺术模板并导出卡片。

核心原则：AI 负责寻找和整理内容，用户负责选择，前端模板负责稳定排版。AI 不直接生成带中文文字的图片，也不自由控制文字坐标。

## 本轮关键决策

- 产品先聚焦古诗词和古文，不做通用长文章工具。
- 不预先建设庞大诗词库，由 DeepSeek 理解用户意图并返回候选。
- 模型原文存在幻觉风险，因此候选标记为“待核验”，用户可以编辑原文。
- 用户直接浏览全部模板，不由 AI 推荐模板。
- 句子数量不向用户暴露，由 AI 与前端根据空间自动决定。
- 所有文字使用固定分区网格和真实尺寸测量，禁止重叠。

## 已实现功能

### 内容流程

1. 自然语言输入：支持诗人、作品、名句和情绪描述。
2. `POST /api/search` 调用 DeepSeek，返回结构化候选作品。
3. 未配置 `DEEPSEEK_API_KEY` 时自动使用本地演示数据。
4. 候选页展示作者、朝代、篇名、推荐理由和首句。
5. 用户选择作品后，可勾选 1-9 条要上卡片的句子并编辑原文。
6. 用户明确勾选所有要放入卡片的句子；未勾选句不会自动补回，数量由前端自动设置布局密度。
7. 生成自定义卡片后，右侧隐藏毛泽东、苏轼、辛弃疾演示切换区，改为显示当前作品和“重新选择诗词”。

### 卡片编辑

- 演示内容集：毛泽东、苏轼、辛弃疾。
- 简体/繁体切换，覆盖正文、来源、主题字和印章。
- 四套背景：纸白、煤墨、铜绿、靛夜。
- 署名统一位于左下角，印章统一位于右下角。
- 大主题字完整居中、极低透明度显示，不被色块遮住或画布裁切。
- 文字块会在作者、模板、背景、简繁切换后重新测量；仅缩小发生溢出的句子。

### 当前模板

1. 编辑留白
2. 碑拓余痕
3. 宋刻手卷
4. 题跋册页
5. 现代构成
6. 活字套印
7. 夜读笺
8. 印象拼贴
9. 字阵残章

“题跋册页”的小框不填充独立背景，只保留淡边界；“现代构成”使用低饱和炭灰和陶土红。

“字阵残章”参考横竖混排文字海报：所有文字簇无框、透明，使用多色层级和淡化的大主题字；通过固定网格保持自由感但避免重叠。

## 技术结构

- `index.html`：当前全部前端、模板 CSS、卡片渲染与内容选择流程。
- `server.js`：无第三方依赖的 Node HTTP 服务、静态文件服务和 DeepSeek 代理。
- `DEEPSEEK_CARD_SPEC.md`：模型内容与卡片输出规范。
- `README.md`：本地运行和 DeepSeek 配置说明。

## 本地运行

```powershell
node server.js
```

访问：`http://127.0.0.1:4174/`

启用 DeepSeek：

```powershell
$env:DEEPSEEK_API_KEY="你的密钥"
$env:DEEPSEEK_MODEL="deepseek-chat"
node server.js
```

API Key 只能保存在服务端环境变量中，不能写入前端。

## 已验证项目

- 前端和服务端 JavaScript 语法通过。
- 演示搜索接口可返回 3 个候选，每篇约 7 条句子。
- 已跑通“搜索 → 候选 → 选句 → 生成卡片”。
- 自定义苏轼卡片显示 7 条句子且无溢出。
- 9 套模板的文字块和主题字边界均经过浏览器测量。

## 下一步建议

1. 使用真实 `DEEPSEEK_API_KEY` 测试模型输出稳定性和错误处理。
2. 接入可靠诗词原文检索来源，对 AI 返回的作者、篇名和原文做校验。
3. 将单文件前端拆分为内容选择、模板配置、渲染器和导出模块。
4. 暂不开发 PNG 导出；使用浏览器截图或打印/PDF保存，避免引入不稳定的 Canvas 渲染依赖。
5. 为 `/api/search` 增加请求频率限制、超时、缓存和日志。
6. 部署前补充移动端模板选择操作与端到端测试。

当前不优先开发登录、社区、完整诗词库和复杂文件上传。
